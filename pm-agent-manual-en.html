PHASE 1 Initial Setup (one time only) Est. 30–60 min
1
Create a Project on Claude.ai
Left sidebar → "Projects" → "New Project"
›

① Open claude.ai
② Click Projects in the left sidebar
③ Click New Project
④ Enter a project name (e.g. "PM Agent") and create it

This project becomes your "PM Agent room." Describe your idea here and it will automatically generate 4 spec files for you.
2
Paste the custom instructions
Project settings → Custom instructions → paste the full file
›

① Open the settings (gear icon) at the top-right of your project page
② Find the Custom instructions field
③ Open custom-instructions-v4.0.txt in a text editor
④ Select all (Ctrl+A → Ctrl+C) and paste it in
⑤ Click Save

Important: You must paste the entire contents of custom-instructions-v4.0.txt. Pasting only part of it will cause incorrect behavior.
3
Upload the knowledge files
Both pm-agent-knowledge-v4.0.md and xp-rules.md
›

① Open the Knowledge section in project settings
② Click Add content → Upload files
③ Upload both of the following files

Files to upload
pm-agent-knowledge-v4.0.md xp-rules.md
xp-rules.md starts nearly empty, but lessons from each project accumulate there over time. Make sure to upload both files.
4
Install Claude Code
Just run 2 commands in the terminal
›

Open a terminal (Mac: Terminal app / Windows: PowerShell) and run the following in order.

Step 1: Check Node.js
node --version

If a version number appears (e.g. v22.1.0) you're good. If nothing appears, install Node.js from nodejs.org first.

Step 2: Install Claude Code
npm install -g @anthropic-ai/claude-code
5
Install global MCPs (shared across all projects)
Context7 and Brave Search — configure once, use everywhere
›

Run each line below one at a time in your terminal.

Context7 MCP (official docs lookup tool)
claude mcp add context7 -s user -- npx -y @upstash/context7-mcp@latest
Brave Search MCP (web search tool) — replace YOUR_API_KEY before running
claude mcp add brave-search -s user -e BRAVE_API_KEY=YOUR_API_KEY -- npx -y @brave/brave-search-mcp-server
How to get a Brave Search API key:
① Go to brave.com/search/api/
② Sign up for the free plan (2,000 queries/month free)
③ Copy your API key and replace YOUR_API_KEY in the command above, then run it
PHASE 2 Describe your idea to the PM Agent (per project) Est. 30 min – 2 hrs
1
Open the project and describe your idea
Just say what you want to build in 1–2 sentences — no special format needed
›

Open your PM Agent project on Claude.ai and send a message like this:

Example message (any wording is fine)
I want to build a budget tracker app where I can log income and expenses and see a monthly graph.
The PM Agent will always start by saying "I will generate 4 output files. Let me ask you some questions first." If spec files appear immediately without any questions, something is wrong.
2
Answer the interview questions (15–40 questions)
Max 3 questions per turn. Most answers are multiple choice
›

The PM Agent interviews you in 4 stages:

STAGE 1Purpose: What are you building? Who uses it? What does success look like?
STAGE 2Scope: What are the minimum features? What's explicitly out of scope?
STAGE 3Behavior: How does each screen work? What happens on errors?
STAGE 4Constraints: External services? Deployment target? Access control?
"I don't know" is a valid answer. The PM Agent will write [ASSUMPTION: ...] and make a provisional decision. You can change it later.
About the circuit breaker: If 3 high-impact unknowns accumulate, the interview pauses automatically. You'll be asked to either narrow the scope or resolve the unknowns before continuing.
3
Receive the 4 output files
Copy the text and prepare to save each file locally
›

After the interview, the PM Agent generates these 4 files:

CLAUDE.mdInstructions for Claude Code (development rules)
vision.mdProduct spec (features, screens, task breakdown)
settings.jsonClaude Code permissions (prevents dangerous operations)
MCP setup commandCommand to add the project-specific tool (Playwright)

Copy the content of each file and save them in the next phase.

PHASE 3 Set up the development environment (per project) Est. 15–30 min
1
Create the project folder
Run these commands manually in the terminal — do NOT ask Claude Code to do this
›
Important: Run these steps manually in your terminal. Do not ask Claude Code to create the folder for you.

You can replace my-app with your preferred project name (letters, numbers, and hyphens only).

Step 1: Create the project
npx create-next-app@latest my-app --typescript --tailwind --eslint --app --src-dir --import-alias "@/*"

A few prompts will appear — press Enter for all of them (accept defaults).

Step 2: Navigate into the folder
cd my-app
Step 3: Initialize UI components
npx shadcn@latest init
Step 4: Initial commit
git init && git add . && git commit -m "initial scaffold"
2
Place the spec files in the project
Save each file the PM Agent generated to the correct location
›

Save the text generated by the PM Agent as actual files in these locations:

CLAUDE.md→ Place directly inside the my-app folder (root)
vision.md→ Place directly inside the my-app folder (root)
settings.json→ Place inside the my-app/.claude/ folder
If the .claude folder doesn't exist:
Open the project in VSCode, create a new folder named ".claude", then create settings.json inside it.
3
Add the Playwright MCP
Required for every project — run inside the project folder
›

Make sure your terminal is inside the my-app folder, then run:

Add Playwright MCP (browser automation tool)
claude mcp add playwright -s project -- npx -y @playwright/mcp@latest
Playwright lets Claude Code control a real browser to verify that your UI actually works. It must be added for each new project.
4
Launch Claude Code and start building
Just type claude in the terminal
›

While inside the project folder (my-app), launch Claude Code:

Launch Claude Code
claude

Once it starts, paste this as your first message:

First instruction (paste into Claude Code)
Read @vision.md and implement the first task in the Task Breakdown.
PHASE 4 Rules during development (every session) Priority: High
1
One task = one session
Finish a task, exit claude, then restart for the next one
›
Claude Code's memory quality degrades as sessions grow longer. Once a feature is complete, exit and start a fresh session for the next task.
Template for starting the next task
Read @vision.md and implement the "(task name)" task in the Task Breakdown.
2
Run /compact every 30 minutes
Memory compression — essential for longer sessions
›

After about 30 minutes of work, type the following in Claude Code:

Memory compression command
/compact
This causes Claude to summarize what it's been doing, freeing up memory for the rest of the session.
3
Always verify the result in your own browser
Never trust "it's done" without checking yourself
›
Important: Claude may say "done" even when things aren't working. Always open http://localhost:3000 in your browser and confirm with your own eyes.

To start the dev server (in a separate terminal window):

Start dev server
pnpm dev
4
If an error persists 3+ times, use the Escape Card
Retrying the same fix over and over makes things worse
›

If the same error hasn't been resolved after 3 attempts, paste this directly into Claude Code:

Escape Card (break out of error loops)
Stop. Follow these steps now: 1. Save the current problem and every fix you've tried into progress.md 2. Tell me to run /clear After /clear, start a new session and enter: "Read @progress.md and solve the problem using a different approach. Do not try any method you already attempted."
5
Add a rule to CLAUDE.md whenever a mistake recurs
Self-improvement to prevent the same error happening again
›

When Claude repeats the same mistake, instruct it to add a prevention rule to CLAUDE.md:

Rule-addition instruction template
To prevent "(describe the problem)" from happening again, add a rule to the last line of CLAUDE.md.
PHASE 5 When a project is complete (capture lessons learned) Makes your next project smoother
1
Run /ev to generate lessons learned
Run this in your Claude.ai PM Agent project
›

Open your Claude.ai PM Agent project and type:

Lesson extraction command
/ev

The PM Agent reviews problems, mistakes, and improvements from this project and outputs them as "XP-XX" format rules.

2
Append the generated rules to xp-rules.md
Open the file in a text editor and paste at the bottom
›

① Copy the rules the PM Agent output (in XP-xx format)
② Open your local xp-rules.md in a text editor
③ Paste at the end of the file and save
④ Max 10 entries — merge or remove older ones if you exceed the limit

xp-rules.md is your personal asset — accumulated AI development experience. Each project makes the next one faster and smoother.
3
Re-upload the updated xp-rules.md to your project
Delete the old file → upload the new one
›

① Open your Claude.ai PM Agent project
② Go to Settings (gear icon) → Knowledge
③ Delete the existing xp-rules.md
④ Upload the updated xp-rules.md

From your next project onward, these lessons will be automatically embedded into every CLAUDE.md the PM Agent generates.
PHASE 6 Routine maintenance (recommended monthly) Keep your knowledge base fresh
1
Run /audit to check for outdated rules
Surfaces stale rules and missing updates caused by tech changes
›

Run this once a month in your Claude.ai PM Agent project:

Audit command
/audit

The PM Agent will propose "rules that are now outdated" and "rules that should be added." Review the suggestions and update pm-agent-knowledge-v4.0.md or xp-rules.md as needed.

If you want to change the tech stack mid-project
① Do not attempt to migrate existing code — starting over is faster
② vision.md can be reused as-is (it's tech-stack agnostic)
③ Ask the PM Agent to "regenerate only CLAUDE.md and settings.json"
④ Start fresh from create-next-app as a new project
