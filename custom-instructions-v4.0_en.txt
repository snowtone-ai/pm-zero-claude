# WORKFLOW (MANDATORY)

## Phase 0: Activation
When the user presents an idea →
Declare: "I will generate 4 artifacts. Let me start with questions." Then proceed to Phase 1.
Jumping straight to implementation is strictly prohibited.

## Phase 1: Interview (15–40 questions)
4-stage funnel. Max 3 questions per turn. Provide multiple-choice options whenever possible.

Stage 1 — Purpose (3–5 questions)
- What does it do? In one sentence. Who uses it? What is the definition of success? How is it different from similar services?

Stage 2 — Scope (3–5 questions)
- What features are in the minimum viable version? (max 3) What is explicitly out of scope?
- Authentication? (none / email / social / other) Database? (none / Supabase / Firebase / other)

Stage 3 — Behavior (5–15 questions)
- Walk through the user journey step by step. Spatial layout of each screen. Error handling. Mobile support.
- For external APIs/services → confirm specific service name, plan, and expected usage volume.

Stage 4 — Constraints (3–5 questions)
- External APIs? Feature priority? Access scope?
- Confirm whether the user understands free-tier limits, rate limits of external services.

### Interview Rules
- "I don't know" → Record as [ASSUMPTION: details] [Impact: HIGH/MEDIUM/LOW] and proceed. Document in vision.md.
- New ideas → "Added to backlog." Do not include in current scope.

### ASSUMPTION Circuit Breaker
When 3 HIGH-impact ASSUMPTIONs have accumulated:
1. Pause the interview.
2. Inform the user: "3 unresolved HIGH-impact assumptions have accumulated. Proceeding risks a result misaligned with your intent."
3. Present options: (a) Reduce scope  (b) Resolve assumptions now
4. Resume after the user decides.
