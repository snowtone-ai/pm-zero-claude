PMエージェント 行動手順書

v4.0 対応 ／ 非エンジニア向け完全ガイド

PHASE 1 初期セットアップ → PHASE 2 インタビュー → PHASE 3 環境構築 → PHASE 4 開発中 → PHASE 5 完了・保存
PHASE 1 初期セットアップ（最初の1回だけ） 所要 30〜60分
1
Claude.ai でプロジェクトを作る
左サイドバーの「Projects」→「New Project」
›

① claude.ai を開く
② 左サイドバーの Projects をクリック
③ New Project ボタンをクリック
④ プロジェクト名を入力（例：「PMエージェント」）して作成

このプロジェクトが「PMエージェントの部屋」になります。アイデアを話しかけると、4つの設計書を自動生成してくれます。
2
カスタム指示を設定する
プロジェクト設定 → Custom instructions に貼り付け
›

① プロジェクト画面右上の 設定（歯車アイコン） を開く
② Custom instructions 欄を見つける
③ お手元の custom-instructions-v4.0.txt をテキストエディタで開く
④ 全文をコピー（Ctrl+A → Ctrl+C）して貼り付け
⑤ Save をクリック

注意： 貼り付ける内容は custom-instructions-v4.0.txt の全文です。一部だけ貼り付けると動作しません。
3
ナレッジファイルをアップロードする
pm-agent-knowledge-v4.0.md と xp-rules.md の2ファイル
›

① プロジェクト設定の Knowledge セクションを開く
② Add content → Upload files をクリック
③ 以下の2ファイルを両方アップロードする

アップロードするファイル
pm-agent-knowledge-v4.0.md xp-rules.md
xp-rules.md は最初ほぼ空ですが、プロジェクトを重ねると教訓が蓄積されていきます。必ず両方アップロードしてください。
4
Claude Code をインストールする
ターミナルで2コマンド実行するだけ
›

ターミナル（Mac：Terminal アプリ ／ Windows：PowerShell）を開いて、以下を順番に実行します。

Step 1：Node.js の確認
node --version

バージョン番号（例：v22.1.0）が表示されればOK。何も表示されない場合は nodejs.org からインストールしてください。

Step 2：Claude Code のインストール
npm install -g @anthropic-ai/claude-code
5
グローバル MCP を導入する（全プロジェクト共通）
Context7 と Brave Search は一度だけ設定
›

ターミナルで以下を1行ずつ実行します。

Context7 MCP（公式ドキュメント参照ツール）
claude mcp add context7 -s user -- npx -y @upstash/context7-mcp@latest
Brave Search MCP（Web検索ツール）※APIキーを差し替えて実行
claude mcp add brave-search -s user -e BRAVE_API_KEY=あなたのAPIキー -- npx -y @brave/brave-search-mcp-server
Brave Search APIキーの取得手順：
① brave.com/search/api/ にアクセス
② 無料プランに登録（月2,000クエリ無料）
③ 発行されたAPIキーをコピーして、上のコマンドの「あなたのAPIキー」部分に置き換えて実行
PHASE 2 PMエージェントにアイデアを話す（プロジェクトごと） 所要 30分〜2時間
1
プロジェクトを開いてアイデアを伝える
「作りたいもの」を1〜2文で自由に話しかけるだけでOK
›

Claude.ai の PMエージェントプロジェクトを開き、以下のように話しかけます：

例文（この形式でなくてもOK）
家計簿アプリを作りたい。収支を入力したら月ごとのグラフが見られるやつ。
PMエージェントは必ず「4つの成果物を生成します。まず質問します。」と宣言してからインタビューを始めます。いきなり設計書が出てきたら、それは正しい動作ではありません。
2
インタビューに答える（15〜40問）
1ターン最大3問。選択肢から選ぶことが多い
›

PMエージェントは4段階に分けて質問します：

STAGE 1目的：何を作る？誰が使う？成功の定義は？
STAGE 2範囲：最小限の機能は？やらないことは？
STAGE 3振る舞い：各画面でどう動く？エラーは？
STAGE 4制約：外部サービスは？公開範囲は？
「わからない」はOKです。 PMエージェントが [ASSUMPTION: ...] と書いて仮決定してくれます。後から変更もできます。
サーキットブレーカーについて： 重要な未確定事項が3つ溜まると自動停止します。「スコープを縮小する」か「今ここで決める」かを選んでください。
3
4つの設計書を受け取る
テキストをコピーしてファイルに保存する準備をする
›

インタビュー完了後、PMエージェントが以下の4つを出力します：

CLAUDE.mdClaude Codeへの指示書（開発ルール集）
vision.mdプロダクト仕様書（機能・画面・タスク一覧）
settings.jsonClaude Codeの権限設定（危険操作の防止）
MCP初期化コマンドプロジェクト専用ツールの導入コマンド

各ファイルの内容をコピーして、次のフェーズで実際のファイルとして保存します。

PHASE 3 開発環境を準備する（プロジェクトごと） 所要 15〜30分
1
プロジェクトフォルダを作る
ターミナルで順番に実行（Claude Codeにやらせない）
›
重要： この手順はターミナルで手動実行します。Claude Codeに「フォルダを作って」と頼まないでください。

my-app の部分はお好みのプロジェクト名に変更OK（英数字とハイフンのみ）。

Step 1：プロジェクト作成
npx create-next-app@latest my-app --typescript --tailwind --eslint --app --src-dir --import-alias "@/*"

途中でいくつか質問が出ます。すべて Enter（デフォルト）でOK。

Step 2：フォルダに移動
cd my-app
Step 3：UIコンポーネント初期化
npx shadcn@latest init
Step 4：最初のコミット
git init && git add . && git commit -m "initial scaffold"
2
設計書ファイルを配置する
PMエージェントが生成した内容をファイルに保存
›

PMエージェントが生成したテキストを、以下の場所にファイルとして保存します：

CLAUDE.md→ my-app フォルダの直下に配置
vision.md→ my-app フォルダの直下に配置
settings.json→ my-app/.claude/ フォルダの中に配置
.claude フォルダが存在しない場合：
VSCode でプロジェクトを開き、新しいフォルダ「.claude」を作成し、その中に settings.json ファイルを作成してください。
3
Playwright MCP を追加する
プロジェクトごとに必要。プロジェクトフォルダ内で実行
›

ターミナルで my-app フォルダに移動した状態で実行します：

Playwright MCP 追加（ブラウザ自動検証ツール）
claude mcp add playwright -s project -- npx -y @playwright/mcp@latest
Playwright は Claude Code が「ブラウザを操作しながら動作確認する」ためのツールです。プロジェクトごとに追加が必要です。
4
Claude Code を起動して開発を開始する
ターミナルで claude と打つだけ
›

プロジェクトフォルダ（my-app）内で Claude Code を起動します：

Claude Code 起動
claude

起動したら、以下をそのままコピーして最初の指示として貼り付けます：

最初の指示（Claude Code への入力）
@vision.md を読んで、Task Breakdown の最初のタスクを実装してください
PHASE 4 開発中のルール（毎セッション） 重要度：高
1
1タスク = 1セッションを守る
1つ終わったら claude を終了、次のタスクで再起動
›
Claude Code はセッションが長くなるほど「記憶の質」が落ちます。1つの機能の実装が完了したら、いったん終了して新しいセッションで次のタスクを始めてください。
次のタスクへ進む時の入力テンプレート
@vision.md を読んで、Task Breakdown の「（タスク名）」を実装してください
2
30分ごとに /compact を打つ
メモリ節約コマンド。長いセッションで必須
›

30分ほど作業が続いたら、Claude Code に以下を入力します：

メモリ圧縮コマンド
/compact
これを打つと Claude が「今何をしているか」を整理して、残りのメモリを効率よく使えるようになります。
3
実装された画面は必ず自分の目で確認する
「できました」という報告だけで信用しない
›
重要： Claude は「できました」と報告しても、実際に動いていないことがあります。必ずブラウザで http://localhost:3000 を開いて、自分の目で確認してください。

開発サーバーを起動するには（別ターミナルで）：

開発サーバー起動
pnpm dev
4
エラーが3回以上続いたらエスケープカードを使う
同じエラーを何度も試させるのは逆効果
›

同じエラーが3回直らない場合、以下を Claude Code にそのままコピーして貼り付けてください：

エスケープカード（エラーループ脱出）
ストップ。今から以下の手順を実行してください： 1. 現在の問題と試した修正を progress.md に保存する 2. 私に「/clear を実行してください」と伝える /clear 実行後、新しいセッションで以下を入力： 「@progress.md を読んで、別のアプローチで問題を解決してください。前回と同じ方法は試さないでください。」
5
同じミスが繰り返されたら CLAUDE.md にルールを追加する
自己改善で再発を防ぐ
›

Claude が同じ間違いをしたら、そのパターンを CLAUDE.md に追記するよう指示します：

ルール追加の指示テンプレート
今起きた「（問題の内容）」が二度と起きないよう、CLAUDE.md の最終行にルールを追加してください
PHASE 5 プロジェクト完了時（教訓を蓄積する） 次の開発をより良くするために
1
/ev コマンドで教訓を生成する
Claude.ai の PMエージェントプロジェクトで実行
›

Claude.ai の PMエージェントプロジェクトを開いて、以下を入力：

教訓抽出コマンド
/ev

PMエージェントが今回のプロジェクトで発生した問題・ミス・改善点を「XP-XX」形式のルールとして出力します。

2
生成された教訓を xp-rules.md に追記する
テキストエディタで開いてコピペ
›

① PMエージェントが出力したルール（XP-xx の形式）をコピー
② ローカルの xp-rules.md ファイルをテキストエディタで開く
③ 末尾に貼り付けて保存
④ 上限は10項目。超えたら古いものと統合する

xp-rules.md は「あなたのAI開発経験が蓄積される個人資産」です。プロジェクトを重ねるほど、次の開発がスムーズになります。
3
更新した xp-rules.md をプロジェクトに再アップロードする
古いファイルを削除 → 新しいファイルをアップロード
›

① Claude.ai の PMエージェントプロジェクトを開く
② 設定（歯車）→ Knowledge を開く
③ 既存の xp-rules.md を削除
④ 更新した xp-rules.md を新たにアップロード

これで次回のプロジェクトから、この教訓が自動的に CLAUDE.md に組み込まれるようになります。
PHASE 6 定期メンテナンス（月1回推奨） ナレッジの鮮度を保つ
1
/audit コマンドで陳腐化チェックをする
技術仕様の変化や古いルールを洗い出す
›

Claude.ai の PMエージェントプロジェクトで月1回実行します：

定期監査コマンド
/audit

PMエージェントが「古くなったルール」「追加すべきルール」を提案します。内容を確認して、必要なら pm-agent-knowledge-v4.0.md か xp-rules.md を更新してください。

技術スタックを途中で変えたくなった場合
① 既存コードの移行は試みない（やり直した方が速い）
② vision.md はそのまま再利用できる（技術スタックに依存しない仕様書）
③ PMエージェントに「CLAUDE.md と settings.json だけ再生成して」と依頼する
④ create-next-app から新規プロジェクトとしてやり直す
