PM Zero Claude

非エンジニアのアイデアを、AI が自律実行できる仕様書セットに変換するシステム。
このマニュアルは、GitHub からダウンロードした直後の状態から読み始めることを想定しています。

v7.2 Claude Opus 4.7 対応 2026-04-19

このシステムについて

PM Zero Claude は 2 つの部分から成ります。

部分場所役割
PM エージェント Claude.ai（ブラウザ） あなたのアイデアをインタビューで整理し、仕様書セットを生成する

v7.2 で何が変わったか（v7.1 使用者向け）

v7.1 で「架空」と誤記されていた Hook イベントの全面修正、Playwright MCP パッケージ名の誤りの訂正、PowerShell 5.1 と 7+ の差異の明記など、公式仕様との乖離を一掃しました。また、インタビュー前に最新情報を自動調査する「Phase 0 リサーチゲート」が追加されました。

項目v7.1v7.2
インタビュー前リサーチなしPhase 0 リサーチゲート（6 項目・省略禁止）
Hook 数9 本10 本（StopFailure 追加・SessionStart matcher に clear 追記）
Playwright MCP パッケージ@anthropic-ai/playwright-slim（存在しない）@playwright/mcp（Microsoft 公式）
MCP 登録コマンドclaude mcp add（CLI バグあり）claude mcp add-json 推奨
成果物構成段階追加 11 ファイルコア 5 + 段階追加 6（bootstrap.ps1 をコアに昇格）
/audit Skill推奨必須 3 Skill に昇格
xp-rules.md独立運用（10 教訓）v7.2 ナレッジに統合・リセット済み
Note v7.2 では Claude.ai の「カスタム指示（Project instructions）」欄は空白のままで構いません。すべての設定は pm-zero-claude-knowledge-v7.2.md に統合されています。

前提条件の確認

セットアップを始める前に、以下がインストールされているか確認してください。

  • Node.js 22 LTS 以上（nodejs.org からインストール）
  • pnpm 10 以上（npm install -g pnpm でインストール）
  • Git（git-scm.com からインストール）
  • Claude Code（npm install -g @anthropic-ai/claude-code でインストール）
  • Claude.ai のアカウント（Pro プラン推奨）
  • Brave Search の API キー（無料枠あり・後述）

確認方法

PowerShell を開いて、以下を 1 行ずつ実行してください。バージョン番号が表示されれば OK です。

node --version
pnpm --version
git --version
claude --version
Windows のみ PowerShell で「実行できません」と表示された場合は、以下を実行してから再試行してください。
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

Claude.ai の準備

プロジェクトの作成

  1. Claude.ai を開く

    claude.ai にアクセスし、アカウントでログインします。

  2. プロジェクトを作成する

    左サイドバーの「Projects」→「+ New project」をクリックします。名前は「pm-zero」などわかりやすい名前にしてください。

  3. ナレッジにファイルをアップロードする

    プロジェクト設定の「Project knowledge」セクションに、以下の 2 ファイルをアップロードします。

    pm-zero-claude-knowledge-v7.2.md
xp-rules.md
  4. カスタム指示は空のままにする

    「Project instructions（カスタム指示）」欄は何も入力しません。v7.2 ではナレッジファイルに設定が統合されているため不要です。

完了の確認 プロジェクトのチャットに「こんなアプリを作りたい」と入力したとき、Claude が「pm-zero-claude v7.2 で成果物を生成します。まず Phase 0 リサーチを実施します。」と返答すれば成功です。

MCP の登録

MCP（Model Context Protocol）は、Claude Code に外部ツールを使わせるための仕組みです。以下の 2 つを登録します。1 度登録すれば、全プロジェクトで使えます。

Windows ユーザーへ注意 claude mcp add -- cmd /c を使うと /c が C:/ に変換されて設定が壊れる既知バグがあります（2026-04-19 時点で未修正）。以下に示す add-json 形式を使ってください。

Context7（公式ドキュメント参照）

Next.js・React・Tailwind などの最新仕様を Claude に参照させます。API キー不要です。

claude mcp add-json context7 -s user '{"type":"stdio","command":"cmd","args":["/c","npx","-y","@upstash/context7-mcp@latest"]}'

Brave Search（Web 検索）

外部サービスの最新 UI 確認・Phase 0 リサーチ・エラー解決に使います。API キーが必要です。

  1. Brave Search API キーを取得する

    api.search.brave.com/app/keys にアクセス → アカウント作成 → API キーを発行します（無料枠：月 2,000 クエリ）。

  2. MCP を登録する

    YOUR_API_KEY を取得したキーに置き換えて実行します。

    claude mcp add-json brave-search -s user '{"type":"stdio","command":"cmd","args":["/c","npx","-y","@brave/brave-search-mcp-server"],"env":{"BRAVE_API_KEY":"YOUR_API_KEY"}}'

登録の確認

claude mcp list

以下のように表示されれば OK です。

context7      (user)
brave-search  (user)

グローバル設定の更新

Claude Code 全体に適用される設定ファイルを更新します。以下をそのままコピーして PowerShell に貼り付けて実行してください。

@'
{
  "env": {
    "CLAUDE_CODE_USE_POWERSHELL_TOOL": "1",
    "CLAUDE_AUTOCOMPACT_PCT_OVERRIDE": "45"
  }
}
'@ | Set-Content "$env:USERPROFILE\.claude\settings.json" -Encoding UTF8

確認します。

Get-Content "$env:USERPROFILE\.claude\settings.json"
設定項目意味
CLAUDE_CODE_USE_POWERSHELL_TOOL Windows の PowerShell でコマンドを実行できるようにする
CLAUDE_AUTOCOMPACT_PCT_OVERRIDE: 45 コンテキストが 45% 埋まったら自動で圧縮する（品質維持のため）
Note 以前の設定に CLAUDE_CODE_DISABLE_ADAPTIVE_THINKING があった場合は削除します。Opus 4.7 では無視される設定です（Opus 4.7 の adaptive thinking は effort パラメータで制御）。Sonnet 4.6 で off にしたい場合のみ有効です。
セットアップ完了 ここまで完了すれば、PM Zero Claude v7.2 を使い始める準備が整いました。

新規プロジェクトを始める

Phase 1：アイデアを仕様書に変換する（Claude.ai）

  1. Claude.ai の pm-zero プロジェクトを開く

    作りたいものを 1〜2 文で伝えます。「家計簿アプリを作りたい」「レシピを保存できるサイトを作りたい」など、曖昧でも大丈夫です。

  2. Phase 0 リサーチを待つ

    PM がインタビュー前に自動でリサーチを行います（フレームワーク最新バージョン・外部 API の仕様・MCP パッケージ名の確認など 6 項目）。リサーチ結果のサマリが表示されたら確認して「次に進んでください」と伝えます。

  3. インタビューに答える

    PM が 4 ステージ（目的→範囲→振る舞い→制約）で 15〜25 問を質問します。1 ターンに 3〜5 問まとめて来ます。わからないことは「わからない」と答えても進められます。

  4. 成果物ファイルを受け取る

    インタビュー完了後、PM がコア 5 ファイル（CLAUDE.md・vision.md・.env.example・settings.json・bootstrap.ps1）を最初に生成します。各ファイルをコピーして保存してください。

Phase 2：プロジェクトフォルダを作る（PowerShell）

まず、PM から受け取った bootstrap.ps1 をプロジェクトフォルダに配置してから実行します。

# フォルダを作成して移動（プロジェクト名を変更してください）
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\.claude\dev\my-app" | Out-Null
Set-Location "$env:USERPROFILE\.claude\dev\my-app"

# 初回のみ：実行ポリシーの許可
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

# PM から受け取った bootstrap.ps1 をここに配置してから実行
.\bootstrap.ps1

bootstrap.ps1 が自動で Next.js セットアップ・shadcn/ui 追加・Git 初期化・フォルダ作成・MCP 登録を行います。手動での個別実行は不要です。

Note v7.2 では Playwright MCP のパッケージが @playwright/mcp（Microsoft 公式）に変わりました。@anthropic-ai/playwright-slim は存在しないため使用しないでください。

Phase 3：残りのファイルを配置する

bootstrap.ps1 の実行後、PM から受け取った残りのファイルを以下の構造で配置します。

my-app/
├── CLAUDE.md　　　　# PM が生成（コア 5）
├── vision.md　　　　 # PM が生成（コア 5）
├── .env.example　　　# PM が生成（コア 5・必要な API キー一覧）
├── .env.local　　　　# .env.example をコピーして値を入力
├── bootstrap.ps1　　　# PM が生成（コア 5）← bootstrap で実行済み
├── .mcp.json　　　　 # bootstrap が生成（プロジェクト MCP）
└── .claude/
　　 ├── settings.json　# PM が生成（コア 5・Hook 10 本含む）
　　 ├── state.md　　　# bootstrap が生成（タスク状態管理）
　　 ├── decisions.md　# bootstrap が生成（決定履歴）
　　 ├── issues.md　　 # bootstrap が生成（エラー記録）
　　 ├── escape.md　　 # bootstrap が生成（緊急時用）
　　 ├── agents/　　　# 専門エージェント 3 体（段階追加）
　　 ├── skills/　　　# 必須 3 + 推奨 3 Skill（段階追加）
　　 └── hooks/　　　# 自動実行スクリプト 10 本（bootstrap が生成）

Phase 4：実装を開始する（Claude Code）

# プロジェクトフォルダで Claude Code を起動
claude

起動後、以下をそのまま入力します。

@vision.md を読んで、@env-setup で必要な外部サービスの設定を開始してください

外部サービスの API キーをすべて取得したら、実装を開始します。

Task Breakdown の最初のタスクを実装してください

日常の開発

毎回の起動

cd "$env:USERPROFILE\.claude\dev\my-app"
claude

起動すると Hook が自動で state.md を確認し、前回の状態から正確に再開します。

中断・再開

Claude が state.md と実際のコードを照合し、現状を報告した上で次のタスクを提案します。

自動で動いている仕組み

以下は何もしなくても自動で動きます。

タイミング自動で行われること
ファイルを保存したときコードの自動整形・型エラーの検出
エラーが発生したときissues.md に内容を自動記録
タスクが完了したときstate.md の状態を自動更新
文脈が 45% を超えたとき自動で圧縮・state.md から状態を復元
セッション開始時state.md と実コードの整合性を確認

エラーが解決しないとき

⚠ エスケープ手順（3 回以上解決しないとき）

同じエラーが 3 回以上解決しない場合、Claude が自動で /escape を起動します。以下の手順に従ってください。

  1. 現在のセッションで /clear を実行

    Claude Code のチャットに /clear と入力します。

  2. Opus 4.7 に切り替える

    /model opus と入力します。

  3. 以下をコピペして入力する

    Claude Code のチャットにこれを貼り付けます。

    @.claude/escape.md と @.claude/state.md を読み、前回と異なるアプローチで問題を解決してください。前回試したアプローチは避けてください。
  4. 解決後は Sonnet に戻す

    /model sonnet と入力します（Opus は消費量が多いため）。

コマンド一覧

Claude Code で使うコマンド（/ から始まるもの）

/resume 中断後の再開時

state.md と実コードを照合し、前回からの変更と次に着手すべきタスクを報告します。

/console-debug ブラウザエラーが解決しないとき

ブラウザの F12（開発者ツール）からエラーをコピーする手順を案内します。エラーを貼り付けると原因を特定します。

/escape 3 回以上エラーが解決しないとき

試行履歴を escape.md に記録し、Opus 4.7 に切り替えるエスケープ手順を案内します。

/ev プロジェクト完了時

エラーログを分析して再発防止ルールを抽出します。README.md も自動生成します。

/audit 月 1 回・必須

ナレッジの陳腐化を確認します。ライブラリのバージョン・MCP パッケージ名・xp-rules.md との整合性をチェックします。v7.2 から必須 Skill に昇格しました。

Subagent の呼び出し（@ から始まるもの）

@pm-zero 仕様の追加・変更時

開発途中で新機能を追加したいとき。インタビューして vision.md の更新案を作ります。

@env-setup [サービス名] 外部サービスの設定時

Supabase・OpenAI・Stripe などの API キー取得手順を最新の UI に基づいて案内します。

@verifier 状態確認時

state.md に記録されたタスク状態と実際のコードが一致しているか確認します。

自己進化サイクル

PM Zero Claude は、プロジェクトを完了するたびに自分自身のルールを更新します。

プロジェクト完了時の手順

  1. Claude Code で /ev を実行する

    エラーログ（issues.md）を元に、再発防止ルールが自動抽出されます。最大 3 件の教訓が xp-rules.md 形式で出力されます。

  2. xp-rules.md を更新する

    出力された教訓を xp-rules.md にコピーします（10 件上限）。

  3. Claude.ai のナレッジを更新する

    更新した xp-rules.md を Claude.ai の pm-zero プロジェクトに再アップロードします。次回から自動で反映されます。

  4. GitHub にプッシュする

    README.md と xp-rules.md をリポジトリに反映します。

    git add .
git commit -m "chore: /ev 教訓を追加"
git push
Note 教訓は「このプロジェクト固有の対症療法」ではなく、「次のプロジェクトでも通用する抽象化されたルール」にすることが原則です。Claude が自動で判断しますが、最終的にはあなたが承認してから追加されます。

v7.2 での xp-rules.md について

v7.2 への移行にあたり、それまでに蓄積されていた 10 件の教訓がナレッジファイル（pm-zero-claude-knowledge-v7.2.md）に統合されました。xp-rules.md は空の状態からリスタートします。次のプロジェクト完了時から再び教訓を蓄積し始めてください。

困ったとき

bootstrap.ps1 を実行したらブロックされた

Windows のセキュリティ機能によるものです。以下のいずれかで解除できます。

# 方法 A：そのファイルだけ解除する（推奨）
Unblock-File -Path ".\bootstrap.ps1"

# 方法 B：ユーザー設定を変更する
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

claude コマンドが認識されない

npm install -g @anthropic-ai/claude-code

インストール後、PowerShell を再起動してください。

MCP が動かない

# 登録状況を確認
claude mcp list

# 再登録（context7）
claude mcp remove context7
claude mcp add context7 -s user -- npx -y @upstash/context7-mcp@latest

Hook が動かない（prettier・型チェックが走らない）

# Hook の登録状況を確認
claude /hooks

# Node.js のバージョンを確認（22 以上が必要）
node --version

state.md と実際の状態がずれている

実際のコードが正しい情報です。@verifier を実行すると、ずれを自動検出して修正案を提示します。

@verifier

45% が圧縮の目安ですが、作業の流れが頻繁に途切れる場合は 50% に上げてみてください。

  }

よくある質問

Q. 非エンジニアでも使えますか？

はい。このシステムは非エンジニアを対象に設計されています。コードを書く必要はなく、Claude.ai でアイデアを話し、Claude Code で指示を出すだけで動きます。

Q. どれくらいのコストがかかりますか？

Claude Pro プラン（$20/月）と Brave Search の無料枠（月 2,000 クエリ）があれば動きます。通常作業は Sonnet 4.6 を使用し、Opus 4.7 はエラー解決時のみ /escape 経由で使用する設計でコストを抑えています。

Q. プロジェクトの途中でアイデアが変わったら？

Claude Code で @pm-zero この機能を追加したい と入力してください。追加のインタビューを経て、vision.md の更新案が作成されます。

Q. 生成されるアプリの技術スタックは？

デフォルトは Next.js 16 / React 19 / TypeScript / Tailwind CSS v4 / pnpm / Vercel です。インタビュー中に変更可能です。

Q. カスタム指示（Project instructions）に何か入れる必要がありますか？

v7.2 では不要です。すべての設定は pm-zero-claude-knowledge-v7.2.md に統合されています。カスタム指示欄は空白にしてください。

Q. /ev と /audit は何が違いますか？

/ev はプロジェクト完了時に「今回の失敗から学ぶ」コマンドです。/audit は月 1 回、ライブラリのバージョンやナレッジが古くなっていないか確認するメンテナンスコマンドです。

Q. GitHub リポジトリはどう管理すればいい？

このリポジトリには pm-zero-claude-knowledge-v7.2.md・xp-rules.md・README.md・manual.html の 4 ファイルだけ管理すれば十分です。プロジェクトのコードは別のリポジトリで管理します。

Q. v7.1 から v7.2 に移行するには？

Claude.ai のプロジェクトナレッジの pm-zero-claude-knowledge-v7.1.md を削除し、pm-zero-claude-knowledge-v7.2.md をアップロードしてください。既存プロジェクトでは .claude/settings.json の SessionStart matcher に clear を追記し、playwright 関連の MCP を @playwright/mcp に修正します。
