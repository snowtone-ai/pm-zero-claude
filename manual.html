v7.3 の変更点

v7.2 から v7.3 への 4 つのアップデート判断

v7.3 ではユーザーから 4 つの改善要望が提示されました。それぞれの採用判断と設計根拠を示します。

提案 01 Permission 質問を一切なくしたい 採用 — 最優先

調査の結果、原因が 2 層に分かれることが判明しました。settings.json の defaultMode: "bypassPermissions" は VSCode 拡張でバグがあり、設定しても質問が来ます（GitHub #34923）。確実に動くのは Claude Code の CLI 起動オプションのみです。

採用した設計： Layer 1 として --permission-mode bypassPermissions CLI フラグで全 Permission 質問を排除。Layer 2 として settings.json の deny リストで rm -rf / git push --force 等の本当に危険な操作だけを物理ブロック。

結果「これまで本当にダメな許可は一度もなかった」という実績と deny リストの組み合わせで、安全性を保ちつつ質問ゼロを実現。
提案 02 プロエンジニアとの協働を前提としたコーディング 採用 — 設計に統合

2026 年に公開された 14 年キャリアのプリンシパルエンジニアによる実証研究（Claude Code vs Codex 120 時間比較）で、「同一セッションで生成したコードは、そのコードのレビューに測定可能に劣る」 という知見が得られています。

採用した設計（3 層）：

パターン設計効果
Fresh-Reviewpr-reviewer に isolation: worktree実装 context と完全分離したレビュー
TDD ファーストtest-writer（RED）と implementer（GREEN）を分離テストが実装に引きずられない
Tier 化 SubagentTier 1（常時 5）+ Tier 2（自動 4）専門家が専門領域だけを担当
提案 03 bootstrap.ps1 が何度やっても動かない — 代替手段を探してほしい 採用 — setup.bat に置換

失敗の原因は PS 実行ポリシー・UTF-8 BOM・SmartScreen の 3 つが同時に作用する構造的な問題でした。PowerShell スクリプトを直す方向性では根本解決になりません。

採用した設計： Windows 10 1803 以降に標準搭載の cmd.exe のみを使う setup.bat に置換。

提案 04 ナレッジファイルは詳しくすべきか、シンプルにすべきか 採用 — 40% トークン削減

Anthropic の公式ドキュメントに「CLAUDE.md は 200 行未満を推奨」と明記されています。仕組み上、CLAUDE.md は system prompt ではなく user message として注入されるため、長くなるほどコンプライアンス率が低下します。ナレッジファイルも同じ原理が適用されます。

判断： 設計経緯・バージョン変更履歴を削除し、ルールとスキーマのみに集中。

v7.2 ナレッジ（920 行 / 43KB）~17,200 tokens
~17,200 tokens
v7.3 ナレッジ（726 行 / 27KB）~10,900 tokens
~10,900 tokens（40% 削減）
結果設計意図は README に移動。ナレッジファイルは毎セッション読み込まれるため、この 40% 削減は全プロジェクトに恒久的に効く。

ファクトチェックで修正した設計誤り

v7.3 では v7.2 の設計を「別の AI が作ったもの」として冷徹に検証しました。

v7.2 で修正が必要だった記述
  • .claude/rules/ の paths: frontmatter を推奨 → バグあり（#17204）。正しくは globs:
  • .claudeignore をネイティブ機能として案内 → npm パッケージ実装。ネイティブ機能ではない
  • settings.json の defaultMode で Permission 排除を約束 → VSCode 拡張でバグあり（#34923）
  • exit 2 = 常にブロック → イベント依存の挙動（UserPromptSubmit ではブロックしない）
  • Hook イベント 26 種 → 公式では 27 種（UserPromptExpansion / CwdChanged / TaskCreated が不足）

前提条件の確認

セットアップを始める前に、以下がインストールされているか確認してください。

  • Node.js 22 LTS 以上（nodejs.org からインストール）
  • pnpm 10 以上（npm install -g pnpm でインストール）
  • Git（git-scm.com からインストール）
  • Claude Code（npm install -g @anthropic-ai/claude-code でインストール）
  • Claude.ai のアカウント（Pro プラン推奨）
  • Brave Search の API キー（無料枠あり・後述）

確認方法

PowerShell を開いて、以下を 1 行ずつ実行してください。バージョン番号が表示されれば OK です。

node --version
pnpm --version
git --version
claude --version
Pro プランの注意点（2026-04 時点） 5 時間の rolling window は claude.ai・Claude Code・Claude Desktop で共有されます。Claude Code のヘビーな作業は claude.ai のクォータも消費するため、/usage コマンドで残量を確認しながら使用してください。

Claude.ai の準備

プロジェクトの作成

  1. Claude.ai を開く
    claude.ai にアクセスし、左サイドバーの「Projects」→「New project」をクリック。
  2. プロジェクト名をつける
    「PM Zero Claude」など分かりやすい名前をつけます。
  3. ナレッジに 2 ファイルをアップロード
    プロジェクト設定の「Knowledge」セクションに以下の 2 ファイルをドラッグ＆ドロップします。

    pm-zero-claude-knowledge-v7.3.md — PM エージェントの知識ベース
    xp-rules.md — プロジェクト横断の実践知（初期は空）
Note 「カスタム指示（Project instructions）」欄は空白のままで構いません。すべての設定は pm-zero-claude-knowledge-v7.3.md に統合されています。

MCP の登録

PowerShell（管理者不要）で以下を実行してください。

Context7（公式ドキュメント参照）

claude mcp add-json context7 -s user '{"type":"stdio","command":"cmd","args":["/c","npx","-y","@upstash/context7-mcp@latest"]}'

Brave Search（Web 検索）

先に Brave Search API で無料の API キーを取得してください。

claude mcp add-json brave-search -s user '{"type":"stdio","command":"cmd","args":["/c","npx","-y","@brave/brave-search-mcp-server"],"env":{"BRAVE_API_KEY":""}}'
Windows の既知バグ（未修正） claude mcp add -- cmd /c ... の形式は使わないでください。/c が C:/ に変換されて設定が破損します（GitHub #20061）。必ず上記の add-json 形式を使ってください。

登録の確認

claude /doctor   # Warning が出ないこと
claude /mcp      # 全サーバーが connected 状態

グローバル設定の更新

PowerShell で以下を実行して、グローバル設定を書き込みます（BOM なし UTF-8 で安全に書き込みます）。

$content = '{"env":{"CLAUDE_CODE_USE_POWERSHELL_TOOL":"1","CLAUDE_AUTOCOMPACT_PCT_OVERRIDE":"45"}}'
$utf8NoBom = [System.Text.UTF8Encoding]::new($false)
[System.IO.File]::WriteAllText(
    "$env:USERPROFILE\.claude\settings.json",
    $content,
    $utf8NoBom
)
設定効果
CLAUDE_CODE_USE_POWERSHELL_TOOLWindows の PowerShell ツールを有効化
CLAUDE_AUTOCOMPACT_PCT_OVERRIDE: 45コンテキスト 45% で自動圧縮（Pro プランの実測値に基づく）

新規プロジェクトを始める

Phase 1：アイデアを仕様書に変換する（Claude.ai）

Claude.ai の PM Zero Claude プロジェクトを開き、作りたいものを 1〜2 文で伝えてください。

（例）写真を撮るだけで食べたもののカロリーを記録できるアプリを作りたい。
スマホのブラウザから使えて、履歴が見られると嬉しい。

PM エージェントが以下の順で進みます：

  1. Phase 0：事前リサーチ
    最新フレームワークバージョン・外部 API の現行仕様・MCP パッケージ名を Web 検索で確認します。古い情報を前提にした仕様書を防ぐためのゲートです。
  2. Phase 1：4 ステージインタビュー
    目的 → 範囲 → 振る舞い → 制約 の順で 15〜25 問インタビューします。1 ターンに 3〜5 問まとめて聞くため、会話のトークンコストを最小化します。
  3. Phase 2-A：最初の 3 ファイルを受け取る
    CLAUDE.md（≤100 行のシンルータ）・vision.md（仕様書）・.env.example（API キー一覧）を受け取り、内容を確認します。
  4. Phase 2-B：残りの設定ファイルを受け取る
    settings.json（権限 + Hook）・setup.bat・.mcp.json を受け取ります。

Phase 2：プロジェクトフォルダをセットアップする

  1. フォルダを作る
    エクスプローラーで作業フォルダを作成します。場所はどこでも構いません。
  2. 受け取ったファイルを配置する
    Claude.ai から受け取った全ファイルをそのフォルダに配置します。setup.bat は必ずフォルダの直下に置いてください。
  3. setup.bat をダブルクリック
    setup.bat をダブルクリックするだけで .claude/ 以下のディレクトリ構造が作成されます。管理者権限不要です。
  4. .env.local に API キーを記入
    .env.example をコピーして .env.local を作り、各 API キーを記入します。

Phase 3：実装を開始する（Claude Code）

プロジェクトフォルダをターミナルで開き、以下を実行します。

claude --permission-mode bypassPermissions
Permission 質問ゼロで起動 このコマンドで起動すると、git push・ファイル書き込み・npm install などの操作で確認ダイアログが出なくなります。settings.json の deny リストが rm -rf 等の本当に危険な操作だけを物理ブロックします。

Claude Code が起動したら、以下を入力して実装を開始します。

@vision.md を読んで、Task Breakdown の最初のタスクを実装してください

最新 UI を Web 検索で確認した上で、ステップバイステップの API キー取得手順を生成します。古い画面説明によるミスを防ぎます。

困ったとき

setup.bat をダブルクリックしてもウィンドウが一瞬で閉じる

正常な動作です。ウィンドウが閉じた後、エクスプローラーで .claude/ フォルダが作成されているか確認してください。

claude コマンドが認識されない

PowerShell を再起動してください。Node.js インストール後は PATH 反映のために再起動が必要です。改善しない場合は npm install -g @anthropic-ai/claude-code を再実行してください。

MCP が動かない

claude /doctor で Warning が出ていないか確認してください。/c が C:/ に変換されているという Warning が出ている場合は、MCP を一度削除して add-json 形式で再登録してください。

Permission 質問がまだ出る

claude --permission-mode bypassPermissions で起動しているか確認してください。settings.json の defaultMode だけでは VSCode 拡張でバグがあり機能しません（GitHub #34923）。

注意 ANTHROPIC_API_KEY 環境変数がセットされていると Claude Code が API 課金モードに切り替わります。Pro プランの容量制限ではなく API の従量課金になるため、意図しない課金が発生する可能性があります。

Hook が動かない（prettier・型チェックが走らない）

以下の順で確認してください：claude /hooks でイベントが登録されているか → .claude/hooks/*.mjs が存在するか → ファイルの先頭に BOM が付いていないか → node --version（22.x 以上推奨）。

state.md と実際の状態がずれている

state.md と実コードが矛盾した場合、実コードが正 です。/resume を実行すると state.md を実コードに合わせて修正します。

コンパクションが頻繁すぎる

45% はアーキテクチャ上の適切な閾値です。頻度を下げたい場合は CLAUDE_AUTOCOMPACT_PCT_OVERRIDE の値を上げてください（ただし上げ方向には内部クランプがある場合があります）。長い会話を 1 つのセッションで続けるより、タスクが完了したら /compact を手動実行してから新しいトピックに入る方が品質・コスト両面で有利です。

よくある質問

Q. 非エンジニアでも使えますか？

はい。PM エージェントとのやり取りはすべて自然言語です。Claude.ai でアイデアを話す → 受け取ったファイルを配置する → setup.bat をダブルクリックする → Claude Code を起動する、の 4 ステップでコーディングが始まります。コードを書く必要はありません。

Q. どれくらいのコストがかかりますか？

Pro プラン（$20/月）のみで追加課金なし。ANTHROPIC_API_KEY を設定しないよう注意してください（設定すると API 従量課金になります）。5 時間の rolling window は claude.ai と Claude Code で共有されるため、/usage でこまめに残量を確認することを推奨します。

Q. プロジェクトの途中でアイデアが変わったら？

Claude.ai の PM エージェントに変更を伝えると、vision.md の該当部分を修正した差分を生成します。state.md と decisions.md も必要に応じて更新します。

Q. globs: と paths: の違いは何ですか？

.claude/rules/ のファイルスコープ指定は globs: を使ってください。paths: はドキュメントに記載があるものの、実際には複数のバグが報告されており（GitHub #17204, #16299）正常に動作しません。

Q. .claudeignore ファイルは使えますか？

Claude Code のネイティブ機能ではありません（機能追加の要望は GitHub #579 で進行中）。ファイルのアクセス制限が必要な場合は、settings.json の deny リストに Read(./.env) などのルールを追加してください。

Q. xp-rules.md は空のままでいいですか？

はじめは空（または初期状態）で構いません。プロジェクトを完了するたびに /ev で教訓を抽出し、承認したものだけ追記していきます。上限は 10 項目です。
