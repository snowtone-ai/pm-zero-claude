# プロジェクト横断教訓（xp-rules.md）
# /ev コマンドで生成されたルールをここに蓄積する。上限10項目。
#

## XP-1: Playwright MCPのトークン膨張

- 発生状況: UI検証のたびにPlaywright MCPでブラウザ操作を実行していた
- 表層の症状: 1テストで89-114Kトークン消費。コンテキスト使用率が急速に増大
- 根本原因: MCPはツールスキーマとスナップショットをコンテキストにインライン展開するため、ブラウザ操作のたびに大量トークンを消費する
- ルール: UI検証時 → playwright-slim（73%スキーマ削減）を使い、1回の検証は10ステップ以内に制限する。検証結果からPlaywrightテストファイルを生成し、以降はHooksで自動実行する

## XP-2: 確定的ルールのCLAUDE.md依存

- 発生状況: prettier適用やlint実行をCLAUDE.mdのルールとして記述していた
- 表層の症状: Claude Codeがprettierやlintの実行をスキップする場合がある
- 根本原因: CLAUDE.mdは確率的に遵守されるため、100%遵守が必要なルールには不適切。さらにルール数が増えると全体の遵守率が均一に低下する
- ルール: 確定的に強制するルール（prettier/build/lint/type-check）→ Hooksに配置する。CLAUDE.mdからは除外し、判断を伴うルールのみCLAUDE.mdに残す
