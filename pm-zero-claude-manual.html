概要

pm-zero-claude は、プログラミング未経験者でも Claude.ai と Claude Code を使って Webアプリを開発できるようにするシステムです。

💡 アイデア → 🎤 インタビュー → 📄 5成果物 → 🤖 Claude Code → 🚀 完成アプリ

5つの成果物

必須 CLAUDE.md

Claude Codeへの指示書。技術ルール・検証手順・エラー対応を記載。30-100行。

必須 vision.md

アプリの設計書。機能・画面・データ・タスク分割を網羅。

必須 settings.json

Claude Codeの権限設定 + 自動品質チェック（Hooks）。

必須 issues.md

エラー自動記録ファイル。プロジェクト完了時の教訓抽出に使用。

必須 初期化手順

MCP導入・プロジェクト作成のコピペ手順。

v6.0 の新機能

  • トークン最適化アーキテクチャ — ツール追加ではなく設計レベルで70-90%コスト削減
  • インタビュー3-5問/ターン — 旧3問/ターンから拡張。75%のインタビューコスト削減
  • フェーズ間圧縮 — 各ステージ完了時に構造化要約を生成。二次関数的コスト膨張を防止
  • 3層検証 — Hooks自動テスト → playwright-slim → ユーザー目視の3段階
  • Hooks大幅強化 — ファイル保存時にbuild/lint自動実行。AIトークンゼロで80%のエラー検知
  • playwright-slim — 公式MCPからトークン73%削減。セットアップ手順の変更なし
  • コンテキスト閾値引き下げ — 60%→50%で予防的にcompact実行

セットアップ

前提条件

  • Claude Pro プラン（月額 $20）
  • Node.js 20.9 以上がインストール済み
  • Claude Code がインストール済み
  • Brave Search API キーを取得済み（無料）

Step 1: Claude.ai プロジェクト作成

Claude.ai にログインし、新しいプロジェクトを作成します。

  1. Claude.ai 左メニューの「Projects」→「Create Project」
  2. プロジェクト名を入力（例：「PM Agent」）
  3. 「Custom Instructions」に、提供されたカスタム指示を貼り付ける
  4. 「Knowledge」に以下の2ファイルをアップロード：
    • pm-zero-claude-knowledge-v6.0.md
    • xp-rules.md

Step 2: グローバル MCP の確認

Context7 と Brave Search は一度だけグローバル導入すれば全プロジェクトで使えます。未導入の場合のみ実行してください。

Context7 MCP（未導入の場合のみ）
claude mcp add context7 -s user -- npx -y @upstash/context7-mcp@latest
Brave Search MCP v2（未導入の場合のみ）
claude mcp add brave-search -s user -e BRAVE_API_KEY=あなたのAPIキー -- npx -y @brave/brave-search-mcp-server
⚠️ 旧パッケージ @modelcontextprotocol/server-brave-search は非推奨です。@brave/brave-search-mcp-server を使ってください。

Phase 1: インタビュー（15-30問）

PMが4段階で質問します。1ターン3-5問、選択肢付き。

Stage内容問数ターン数
1. 目的何をする？誰が使う？成功の定義は？3-51
2. 範囲最小機能は？認証は？DBは？3-51
3. 振る舞い画面遷移、レイアウト、エラー処理5-152-4
4. 制約外部API、優先順位、公開範囲3-51
💡 「わからない」と答えてOK。PMが仮決定し、影響度を記録します。

v6.0変更：各Stageの終了時に、PMが収集情報を構造化要約にまとめて提示します。これにより会話履歴が圧縮され、トークンコストが75%削減されます。

Phase 2-4: 成果物生成 → 品質チェック → ハンドオフ

インタビュー完了後、PMが5つの成果物を自動生成します。21項目の品質チェックを通過した後、セットアップ手順と一緒に提供されます。

あなたがやることは「成果物をファイルとして保存する」だけです。

成果物の詳細

CLAUDE.md — Claude Code への指示書

Claude Code が従うルールを記載したファイル。プロジェクトルート直下に配置します。

  • 30-100行が最適（200行以内厳守）
  • 全ルールが「〜の時 → 〜する」形式
  • 技術スタック宣言、地雷回避、検証手順、エラー対応を含む
💡 Claude Code は CLAUDE.md を100%守るわけではありません。v6.0ではprettier/build/lint等の重要ルールをHooks（settings.json内）で確定的に強制します。CLAUDE.md には判断を伴うルールのみ記載します。
vision.md — アプリの設計書

7セクションからなる完全な設計書です。

  • Purpose: アプリの1文定義、対象ユーザー、解決する課題
  • Features: 各機能の受入基準（Given/When/Then）
  • Pages & UI: 全画面のレイアウト
  • Data Model: データの構造と関係
  • Task Breakdown: 実装タスク（1タスク＝1文・1成果物・1検証）
  • Out of Scope: やらないこと
  • Assumptions: 仮決定事項
settings.json — 権限設定 + 自動品質チェック

Claude Code の操作権限を制御します。.claude/settings.json に配置します。

権限対象意味
deny.env, rm -rf, sudo, curl 等絶対に実行しない
ask設定ファイル変更実行前に確認する
allowsrc/, public/, issues.md 等自由に実行する

v6.0新設：自動品質チェック（Hooks）

settings.json 内の Hooks が以下を自動的に実行します。あなたは何もする必要ありません。

  • ファイル保存時 → prettier で自動整形 + ビルドエラー即時検知
  • コミット前 → lint + 型チェック（エラーがあればコミットを阻止）

これにより、エラーの約80%がAIトークンを消費せずに検知されます。

issues.md — エラー自動記録

Claude Code がエラーや手戻りを自動的に記録するファイルです。

  • エラー発生時: 日時・タスク名・エラー内容・修正方法・結果を記録
  • 手戻り発生時: 原因と対処を記録
  • /ev コマンド実行時にこのファイルを参照して教訓を抽出

あなたは何もする必要ありません。Claude Code が自動で書き込みます。

/clear 実行後、新しいセッションで以下を入力：

トラブルシューティング

Claude Code が CLAUDE.md のルールを無視する

CLAUDE.md は確率的に無視されることがあります（仕様）。対策：

  • ルールを「〜の時 → 〜する」形式にする
  • CLAUDE.md を短くする（行数が多いほど無視率上昇）
  • 100%遵守が必要なルールは Hooks（settings.json内）で強制する
  • v6.0では prettier/build/lint を Hooks に移動済み。残るルールの遵守率が向上
shadcn init で質問が表示される

-d フラグを付けてください：npx shadcn@latest init -d

これでデフォルト設定が自動適用され、質問は表示されません。

&& でコマンドが動かない

シェルの種類によっては && が機能しません。コマンドは1つずつ実行してください。pm-zero-claude v6.0 では全ての初期化手順が個別コマンドになっています。

コンテキストが足りなくなる

/context で使用率を確認。50%超なら /compact を実行（v6.0で閾値を60%→50%に引き下げ）。それでも不足する場合は /clear して新しいセッションで再開。

技術スタックを変えたい

既存コードの移行は非推奨です。vision.md はそのまま再利用できるので、CLAUDE.md と settings.json だけ再生成して create-next-app からやり直してください。

playwright-slim が動かない

公式Playwright MCPにフォールバックしてください：

playwright-slim を削除
claude mcp remove playwright -s project
公式 Playwright MCP を導入
claude mcp add playwright -s project -- npx -y @playwright/mcp@latest
Brave Search MCP が動かない

旧パッケージを使っている可能性があります。以下で更新してください：

旧パッケージを削除
claude mcp remove brave-search -s user
v2 を導入
claude mcp add brave-search -s user -e BRAVE_API_KEY=あなたのAPIキー -- npx -y @brave/brave-search-mcp-server
Hooks のビルドチェックが遅い

PostToolUse の build チェックはファイル保存のたびに実行されます。プロジェクトが大きくなるとビルド時間が長くなる場合があります。対策：

  • settings.json の Hooks から build チェックを一時的にコメントアウトする
  • コミット前の PreCommit Hook は残す（最低限の品質ゲート）
